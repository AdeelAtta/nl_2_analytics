## Spikes

### SPIKE-001: Context Layer Accuracy
- **Hypothesis**: Hybrid retrieval (dense vector + keyword BM25 + graph traversal) achieves > 0.8 precision@10 for context retrieval on enterprise schemas without human annotation.
- **Experiment**: Build a test harness with 3 enterprise schema samples (100-500 tables each). Create 50 benchmark queries with annotated relevant schema context. Run 3 retrieval strategies (vector-only, hybrid, hybrid+graph). Measure precision, recall, and context window usage.
- **Success criteria**: Precision@10 > 0.8 for at least 2 of 3 schema samples. Context window usage < 60% of available tokens.
- **Timebox**: 2 weeks.

### SPIKE-002: Cold-Start Strategy
- **Hypothesis**: A cold start bootstrap using synthetic DDL descriptions generated by LLM + schema element embeddings from public SQL corpuses produces acceptable baseline accuracy (> 40% execution accuracy on simple queries).
- **Experiment**: Create 3 cold-start scenarios (no prior data, only DDL, DDL + 50 example queries). For each, run the full NL2SQL pipeline. Measure execution accuracy improvement from each data increment.
- **Success criteria**: DDL-only baseline achieves > 30% accuracy. DDL + 50 queries achieves > 40% accuracy. Each increment shows measurable improvement.
- **Timebox**: 2 weeks.

### SPIKE-003: Model Router Accuracy
- **Hypothesis**: A lightweight classifier combining query length, number of tables referenced, join complexity, and aggregation patterns can predict query complexity class (simple/medium/complex) with > 90% accuracy, enabling correct model tier selection.
- **Experiment**: Collect 500 labeled queries (SQL + complexity class). Build a heuristic classifier + optional lightweight ML classifier. Measure accuracy, precision, recall per class.
- **Success criteria**: Accuracy > 90%. No complex query classified as simple. Simple queries sent to correct tier > 95%.
- **Timebox**: 1 week.
